How to release a new version of IT++

This document describes briefly the recommended procedure of releasing a new version of IT++ using SourceForge project site.

  • Update the NEWS file with an appropriate information what has changed since the previous release
  • Tag CVS sources with the release name, e.g.:
    % cvs tag release-3-8-0-rc1
  • Check out a clean working copy using the release tag:
    % cvs -d :ext:developer@cvs.sourceforge.net:/cvsroot/itpp checkout -- -r release-3-8-0-rc1 -d itpp-3.8.0-rc1 itpp
  • Regenerate configure, Makefile.in and other autogenerated files using % ./autogen.sh shell script. To perform this step, autoconf, automake and libtool tools has to be available in the system.
  • Configure the updated sources:
    % ./configure
  • Create the distribution package using % make distcheck command. This will configure, compile and check the package. The output package will be named itpp-3.8.0-rc1.tar.gz, for instance.
  • Upload the distribution package to Sourceforge:
    • Start ftp connection with the SourceForge upload server:
      % ftp upload.sourceforge.net
    • Login as anonymous and give email as password
    • Change remote directory to /incomming:
      % cd /incoming
    • Upload the package:
      % mput itpp-3.8.0-rc1.tar.gz
  • Access SourceForge file release interface and add the package. Updates may take 30 minutes!
  • Update the online HTML documentation if necessary:
    • Move and rename the locally generated HTML docuementation using the following command:
      % mv doc/html itpp-html-doc-3.8.0-rc1
    • Create a tar.gz package with the documentation:
      % tar zcf itpp-html-doc-3.8.0-rc1.tar.gz itpp-html-doc-3.8.0-rc1
    • Upload the documentation package into the SourceForge server:
      % scp itpp-html-doc-3.8.0-rc1.tar.gz developer@shell.sourceforge.net:/home/groups/i/it/itpp/htdocs/doc/
    • Login into the SourceForge shell account:
      % ssh developer@shell.sourceforge.net
    • Change the current directory:
      % cd /home/groups/i/it/itpp/htdocs
    • Unpack the uploaded documentation:
      % tar zxf doc/itpp-html-doc-3.8.0-rc1.tar.gz
    • Rename the unpacked directory to 3.8.0-rc1:
      % mv itpp-html-doc-3.8.0-rc1 3.8.0-rc1
    • Update the latest link:
      % rm latest; ln -s 3.8.0-rc1 latest
    • Change the group properities of the uploaded files:
      % chmod -R g+w 3.8.0-rc1 doc/itpp-html-doc-3.8.0-rc1.tar.gz
  • Send an announcement to: itpp-announce@lists.sourceforge.net
